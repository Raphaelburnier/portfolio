About

Analyze, model, deploy.

I'm 19, based in Lausanne (Swizerland). Full-stack developer by background, currently adding data science to that with a certification. Most of what's here comes out of that combination: I build the interface and the backend, then spend the rest of the time making sure the model underneath actually holds up.

In practice that means React on the front, Python and FastAPI underneath, and increasingly some machine learning holding it together. Right now the certification runs alongside side projects that stretch what I actually know: a piano app that turns MIDI into something playable on screen, a forecasting model for grid carbon intensity, an agent-driven coding workflow that runs itself while I sleep.

I'm not chasing the newest framework for its own sake. I like tools that work, and I like understanding why they work. Most of what's here started as "can I actually build this" more than "should I build this." That same itch is behind a video game I'm building, set in a far-off future I've been sketching out for a while, since games are what got me into code in the first place. Away from the screen, I play classical piano, mostly Chopin, and I spent six years on the drums, which is its own kind of debugging.

Selected projects

A few builds I'm proud of.

01
Chrome Extension Claude API Notion API Obsidian API CSS templates

Notion Web Clipper

A Chrome extension that captures web content, text or screenshot, sends it to the Claude API to turn it into structured Markdown, and files it on the right Notion page.

What sets it apart is the pairing of CSS templates with Claude generation. The core challenge is routing: Claude ranks existing Notion pages by confidence and suggests a destination, which the user confirms before anything is written. Runs BYOK with a strict JSON schema in the prompt and Haiku for cost and latency.

02
PyTorch Transfer learning EfficientNet Computer Vision Python

Skin Lesion Classifier

A personal ML project that classifies dermoscopic skin lesions as benign or malignant, trained on the BCN20000 dataset (18,946 images). The approach is transfer learning: fine-tuning a pre-trained EfficientNetB0, with an end-to-end pipeline (stratified 70/15/15 split, custom Dataset, DataLoaders, training loop with early stopping).

It's a learning project as much as a results one: I built it to get hands-on with PyTorch coming from a Keras/TensorFlow background. Current results sit around 82% validation accuracy and 75% on test.

03

StoriA

StorIA writes short children's stories from a few prompts: a theme, a couple of characters, sometimes a lesson to sneak in. Streamlit handles the interface, FastAPI does the actual generation work, and the whole thing runs on GCP. I built it mostly to see how far you can push prompt engineering before you actually need fine-tuning.

Piano App

PianoApp turns sheet music or a recording into a Synthesia-style falling-blocks display, so you can see what to play instead of reading a score. React Native handles the frontend, FastAPI runs the backend, and Basic Pitch / MT3 do the actual audio-to-notes transcription.
